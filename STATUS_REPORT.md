# ElectionPulse 현재 상태 보고서 (2026-04-04)

## 1. 프로젝트 개요

선거 캠프용 AI 분석 SaaS 플랫폼.
고객이 가입하면 자동으로 뉴스/유튜브/커뮤니티/검색트렌드를 수집하고,
AI가 분석하여 전략을 제안하는 프로그램.

기술 스택: FastAPI + PostgreSQL + Redis + Next.js + Celery + Claude CLI

## 3. 서버 실행 방법

```bash
# Docker (PostgreSQL + Redis)
cd /Users/jojo/pro/mybot/docker
docker compose up -d postgres redis

# 백엔드
cd /Users/jojo/pro/mybot/backend
source .venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8100 &

# Celery Worker + Beat
celery -A app.collectors.tasks:celery_app worker --loglevel=info --concurrency=2 &
celery -A app.collectors.tasks:celery_app beat --loglevel=info &

# 프론트엔드
cd /Users/jojo/pro/mybot/frontend
npx next dev -p 3100 &
```

포트: PostgreSQL 5440, Redis 6380, Backend 8100, Frontend 3100

## 6. 🔴 심각한 문제점 (즉시 수정 필요)

### 6-1. 프론트엔드 데이터 표시 안 됨
- 대시보드에 데이터가 느리게 로딩되거나 빈 화면
- 여론조사 페이지 에러 발생
- 차트가 기본 수준 — 프로답지 않음
- 우리 후보가 항상 첫 번째로 안 나오는 경우 있음
- 뉴스 날짜가 기사 날짜가 아닌 수집 날짜로 표시

### 6-2. 데이터 품질
- collected_at이 NULL인 데이터 존재 (수동 INSERT 시 누락)
- 여론조사 중복 (같은 조사가 2번 저장됨)
- 김성근 동명이인 필터가 윤건영 캠프에는 안 걸려있음
- 감성 분석 정확도 — AI 분석 안 쓰면 키워드 기반이라 부정확

### 6-3. 온보딩 후 초기 수집 미실행
- 회원가입 → 온보딩 완료 후 자동 수집이 안 됨
- 사용자가 "지금 수집" 버튼을 눌러야 첫 데이터 생김
- 스케줄은 설정되지만 다음 스케줄 시간까지 데이터 없음

### 6-4. 네이버 API 401 문제
- NAVER_CLIENT_ID/SECRET이 만료되었거나 인증 안 됨
- 뉴스/블로그/카페 전부 스크래핑 폴백으로 동작
- 스크래핑은 네이버가 차단할 수 있음 (403 에러 간헐 발생)

## 7. 🟡 중요한 미완성 기능

### 7-1. SNS 수집 미구현
- 인스타그램, X(트위터), 페이스북 수집 없음
- API 제한이 크지만 해시태그 추적은 가능
- 선거 캠프에서 SNS 모니터링은 필수

### 7-2. 키워드 분석 도구 미완성
- 키워드별 검색량 조회 API는 있지만 프론트 UI 부실
- 실시간 검색어 순위 없음
- 키워드 추가 → 검색량 확인 → 해시태그 추천 흐름 안 됨
- 프로토타입에 있던 블로그 태그 검색량 연동 안 됨

### 7-3. 과거 선거 심층 분석 미완성
- 데이터는 있지만 (선관위 148건, 투표율 21건, 정치환경 5건)
- AI 심층 분석 없음 (패턴 분석, 진보/보수 교대 등)
- 프론트 페이지가 데모 데이터로 되어 있음

### 7-4. 보고서 PDF 생성 미구현
- 텍스트 보고서는 생성되지만 PDF 다운로드 안 됨
- 프로토타입에는 election_pdf.py가 있었음

### 7-5. 콘텐츠 생성 도구 미완성
- AI 블로그 초안 생성 API는 있지만 프론트 연결 안 됨
- 검색량 기반 주제 추천이 실제로 안 보임
- 선거법 체크 기능은 있지만 UI 기본 수준

## 10. 범용화 전략

- 뉴스/유튜브/검색어/커뮤니티 수집 → 후보 이름만 있으면 동작

- 심층 분석 → AI가 작성 + 관리자 검수 → 저장

## 14. 핵심 교훈

1. 기능을 많이 만드는 것보다 각 기능이 제대로 동작하는 것이 중요
2. 하나 만들고 → 검증하고 → 다음으로 넘어가야 함
3. 데모 데이터는 독 — 실데이터만 사용해야 함
4. 프론트엔드 검수를 반드시 해야 함 (API 작동 ≠ 화면 작동)
5. DB에 데이터 넣을 때 NULL 체크 필수
6. 계정 구조는 처음부터 제대로 설계해야 함
