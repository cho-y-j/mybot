# ElectionPulse 품질 완성 플랜

## 현재 문제 진단

### 🔴 작동 안 되는 것
1. 관리자 패널 — 고객 목록만 보이고 실제 관리 불가
2. 김진균 테스트 계정이 수퍼관리자에 회원으로 안 보임
3. 대시보드 — 데모 데이터와 실데이터 혼재
4. 여론조사 페이지 — 프론트 미완성
5. 과거선거 페이지 — 데모 데이터만
6. 검색 트렌드 — 실데이터 API 있는데 UI에 안 붙음
7. 보고서 페이지 — 생성/보기 제대로 안 됨

### 🟡 작동하지만 품질 낮은 것
1. 차트 — 기본 수준, 프로답지 않음
2. 뉴스 목록 — 링크/요약 있지만 UI 투박
3. 콘텐츠 도구 — 동작하지만 디자인 미흡
4. AI 챗 — 동작하지만 응답 느림

### ✅ 잘 되는 것
1. 회원가입/로그인 — JWT 작동
2. 뉴스 수집 — 네이버 34건+ 실데이터
3. 텔레그램 — 연결/발송 작동
4. 온보딩 — 4단계 위저드 작동
5. 선관위 API — 82건 선거 데이터
6. 여론조사 DB — 1,346건 + 충북 6건
7. 스케줄 — 생성/관리 작동
8. AI 챗 — Claude CLI 실제 분석

## 실행 플랜 (순서대로)

### Step 1: 관리자 + 회원 시스템 완성
- [ ] 관리자 대시보드에 실제 회원 목록 표시
- [ ] 회원 상세 (요금제, 선거, 수집현황)
- [ ] 관리자가 회원 대신 설정 가능
- [ ] 김진균 테스트가 정상 회원으로 표시

### Step 2: 대시보드 실데이터 100% 전환
- [ ] 데모 데이터 생성 로직 제거
- [ ] 뉴스 현황 — DB 실데이터
- [ ] 감성 분석 — 실제 수집 뉴스 기반
- [ ] 검색 트렌드 — 네이버 DataLab 실데이터
- [ ] 여론조사 — DB 실데이터 (충북 6건)
- [ ] AI 인사이트 — 실데이터 기반 자동 생성
- [ ] 알림 — 실제 위기/기회 감지

### Step 3: 각 분석 페이지 실데이터 연동
- [ ] 뉴스 분석 — 수집된 37건 실뉴스
- [ ] 여론조사 — 충북 6건 + 수동 입력
- [ ] 검색 트렌드 — DataLab 실시간
- [ ] 과거선거 — 선관위 148건
- [ ] 후보 비교 — 실데이터 기반

### Step 4: UI/UX 프로페셔널 리디자인
- [ ] 대시보드 레이아웃 정리
- [ ] 차트 리디자인 (그라디언트, 애니메이션)
- [ ] 카드 디자인 통일
- [ ] 모바일 반응형 확인
- [ ] 로딩/에러 상태 처리

### Step 5: 전체 통합 테스트
- [ ] 새 계정 가입 → 온보딩 → 수집 → 분석 전체 플로우
- [ ] 관리자 패널에서 회원 관리
- [ ] 텔레그램 보고서 발송
- [ ] AI 챗 실데이터 응답
